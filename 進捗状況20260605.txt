▽src フォルダ
・src/app.py
カルテ作成ツールのGUIアプリ本体。

PySide6で作られたデスクトップアプリ。
ボタン操作で以下を実行するための画面。

DBonk取得
summary作成
カルテPNG作成
ログ表示
設定ファイル編集

背景画像やアイコンは ui/assets を参照している。


・src/scraper.py
DBonkからギルドデータを取得する本体。

PlaywrightでDBonkにログインし、Guild Rankingからギルドを検索して、詳細ページから以下を取得する。

members
summary
node_history

保存先：

data/ギルド名/guild_ギルド名_日付.xlsx
data/bdm_guild.sqlite3

現在の状態では、DBonk取得処理は維持されていて、Excel保存後にSQLite保存も行う。
save_guild_snapshot_to_sqlite() が追加済みで、Excel保存後に呼び出される。

重要な処理：

DBonkログイン
Asiaサーバー選択
Guild Ranking検索
ギルド詳細ページ遷移
Active Guild Members取得
summary取得
node_history取得
Excel保存
SQLite保存
src/db.py

SQLite保存用の共通処理。

DBの場所：

・data/bdm_guild.sqlite3

作成・管理するテーブル：

guild_snapshots
member_snapshots
import_files
node_history_snapshots
guild_summary_snapshots

保存される情報：

ギルド単位の取得履歴
メンバーごとのCPM/FCP/レベル
summary情報
node_history情報
既存Excelの取り込み済み管理

member_snapshots には、将来のクラス名対応用に以下の枠あり。

class_name_raw
class_name_normalized
class_name_version

今はクラス名の抽出は未実装。
新クラス名が実装されてDBonk表示が安定したら対応予定。

src/import_excel_to_db.py

既存ExcelをSQLiteへ取り込む移行用スクリプト。

対象：

data/**/guild_*.xlsx

取り込むシート：

members
summary
node_history

使い方：

python -m src.import_excel_to_db

目的：

過去に作ったExcelをSQLiteにまとめて入れる

import_files で取り込み済みを管理する。
file_pathは絶対パスではなく、原則としてプロジェクトルートからの相対パスで保存する方針。

src/analyze.py

ギルドExcelを読み込んでsummaryを作る分析処理。

入力：

data/ギルド名/guild_ギルド名_日付.xlsx

出力：

analysis/summary_日付.xlsx

計算している内容：

平均CPM
総CPM
人数
CPM帯ごとの人数
中央値
順位
上位依存度
層の厚さ
伸び傾向
自動コメント

現時点では Excelベースの分析。
SQLite読み込みにはまだ完全移行していない。

src/make_card.py

カルテPNG生成処理。

入力：

analysis/summary_日付.xlsx
data/ギルド名/guild_ギルド名_日付.xlsx
template/karte.xlsx

出力：

output/cards/ギルド名/karte_ギルド名_日付.png

主な処理：

summaryからギルド情報を読む
ギルドExcelからメンバー一覧を読む
template/karte.xlsx のプレースホルダーを置換
メンバー欄を書き込み
グラフ作成
Excel経由でPNG出力
src/cleanup_outputs.py

古いカルテPNG整理用スクリプト。

目的：

カルテPNGは最新以外いらないので整理する

使い方：

python -m src.cleanup_outputs --dry-run
python -m src.cleanup_outputs --keep-latest-per-guild

デフォルトは、ギルドごとに最新1枚を残す方針。

重要：

PNGを削除してもSQLiteデータは消えない

bot/cogs/admin.py

Bot投稿ログ整理用コマンド。

/clean_bot_logs

bot/cogs/ai_talk.py

MぼっとAIトーク機能。

コマンド：

/talk

現在の主な仕様：

mode: chat / translate
language: JP / EN / FR / ZH / RU / ES / AR
通常chatは会話履歴を保持
translate modeは履歴保存しない
ユーザーID＋チャンネルIDごとに履歴管理
直近履歴＋簡易要約メモリ
回数制限・クールダウンあり

翻訳や多言語会話は一応実装済みだが、使用モデルが軽量なため精度はまだ弱め。
将来的に高性能モデルや翻訳専用モデルへ変更予定。
